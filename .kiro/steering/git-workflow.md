# Git運用ルール

## ブランチ戦略

- 基本的にはfeatureブランチを作成して作業する
- ブランチ名は `feature/` プレフィックスをつける（例: `feature/add-user-api`）
- ユーザーから明示的にmasterへの直接コミット・プッシュを指示された場合は、そのまま従う

## コミット・プッシュの手順

1. 現在のブランチを確認する
2. ユーザーからmasterへの直接コミットの指示がない場合、masterにいればfeatureブランチを作成してチェックアウトする
3. 変更をコミットする
4. 該当ブランチにプッシュする

## コミットメッセージ

- 日本語で記述する
- 簡潔に変更内容を表現する
